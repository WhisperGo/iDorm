notes:

Konsep: Bot tidak belajar real-time. Tapi bot mencatat (logging) semua percakapan di mana dia bingung (Confidence Score rendah).

Siklus Kerja (Cycle):

Interaksi: User chat.

Filter: Jika confidence < 0.60, simpan teks tersebut ke database khusus (tabel failed_predictions).

Labeling (Tugas Anda): Seminggu sekali, Anda buka tabel itu. Anda lihat: "Oh, banyak yang ngetik 'peminjaman batal'. Bot bingung." -> Anda beri label manual: cancel_booking.

Retrain: Masukkan data baru itu ke training_dataset.csv dan latih ulang (train_model.ipynb).

Deploy: Update model .pkl di server.

Kelebihan: Model makin pintar secara terkontrol. Tidak ada risiko error.
